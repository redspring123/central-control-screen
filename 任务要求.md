## 1. 前端开发任务

### 1.1 S2主控屏界面
目前需要完成的部分：
- 左上角预留用户信息界面跳转入口
- 主体分为两条竖列
- 左部分为每行两个向下排列的小卡片，主要用于实现简单全屋控制，其次在下方预留运动健康功能的入口，一键呼叫功能也要预留接口
- 右部分为每行两个向下排列的大卡片，最多显示量为六个，依次为“家庭能耗”“居家康养”“家庭安防”“家庭网络”“在线问诊“和”可视对讲“卡片，其中居家康养和在线问诊功能要提供跳转入口

### 1.2 居家康养（一键呼叫）服务卡片
目前需要完成的部分：
- 在右上角预留用户信息简单显示
- 主体分为左侧的个人健康信息显示（和下方的“一键呼叫”大按钮），右侧为各种功能卡片，排列方式同为两列向下排列
- 右侧的在线问诊和运动健康功能需要预留跳转接口
- 左侧的个人健康信息展示在顶部需要显示个人信息与医生信息。下方的人物3d模型可以暂时不实现，要提前预留各种健康信息的接口，如“体温”“血糖”等等。
- ”一键呼叫“功能需要预留拨号功能的接口，在点击后按钮颜色变化。

### 1.3 运动健康功能卡片
>由主界面或居家康养卡片内的运动健康功能跳转过去
目前需要完成的部分：
- 主要分为上下两部分
- 上部分由两个圈图表示步数与热量消耗的进度图组成，需要预留步数与热量的数据接口
- 下部分由每行最多4个卡片，最多2列向下排列，只需要简单展示基础健康数据即可（如果可以，在右方可以配参考图）

### 1.4 在线问诊功能卡片
>由主界面或居家康养卡片内的在线问诊功能跳转过去
目前需要完成的部分：
- 整体为筛选部分和显示部分组成
- 筛选部分需要实现关于医院与科室的筛选（含有重置条件按钮）
- 卡片展示最大数量为每行四个最多八个

### 1.5 一些细节
- 如果感觉我写的不够详细，可以自己添加细节，然后尽可能仿照参考页面的图片进行设计
- 整体页面的设计建议使用暖色调或更加柔和的色调，契合康养系统的需求
- 字体尽量调大，符合康养人群习惯，必要时还需要设置语音播报功能，当然语音操作功能估计没机会实现了

## 2. 后端开发任务

### 2.1 接口调用需求
鸿蒙集成了一套轻量化的数据与存储方案，需要各位负责后端的同学查找相关文档，目前可能涉及的接口有：
- 数据存储
- 传感器数据接口
- 账号管理
- 电话通讯

### 2.2 关于前后端对接
建议采用优先前端的顺序，后端根据前端设计的数据结构来处理数据（和算法逻辑）。建议前端开发同学和后端开发同学积极交流，统一结构（可以考虑建立一个共享文档）。
这里的数据结构指类似以下代码这种结构，再比如设计各种卡片时，每个卡片都会携带一个列表的信息。
``` json
//仅供参考
{
    uid : int
    username : string
    password : string
}
```

### 2.3 关于分工

目前暂且需要两位后端各自认领两个页面负责，完成页面的基础功能。
后续可能会派发新任务，比如各种卡片功能的逻辑实现、软件的打包等等。
